 <span data-ttu-id="cf07b-108">トレースは、アプリケーションで問題が検出された後に有効にし、その問題が解決された後は無効にすることができます。</span><span class="sxs-lookup"><span data-stu-id="cf07b-108">Tracing can be enabled after a problem is detected with the application and then disabled once the problem is resolved.</span></span>  
  
## <a name="enabling-tracing"></a><span data-ttu-id="cf07b-109">トレースの有効化</span><span class="sxs-lookup"><span data-stu-id="cf07b-109">Enabling Tracing</span></span>  

 <span data-ttu-id="cf07b-110">メッセージ フローのトレースを有効にするには、次の例に示すように、.NET Framework 4 の `messageFlowTracing` 構成要素を `true` に設定します。</span><span class="sxs-lookup"><span data-stu-id="cf07b-110">You can enable message flow tracing by setting the .NET Framework 4 `messageFlowTracing` configuration element to `true`, as shown in the following example.</span></span>  
  
```xml  
<system.servicemodel>  
  <diagnostics>  
    <endToEndTracing propagateActivity="true" messageFlowTracing="true" />  
  </diagnostics>  
</system.servicemodel>  
```  
  
> [!NOTE]
> <span data-ttu-id="cf07b-111">`endToEndTracing` 構成要素は Web.config ファイル内にあるため、ETW と同じ方法で動的に構成することはできません。</span><span class="sxs-lookup"><span data-stu-id="cf07b-111">Because the `endToEndTracing` configuration element resides in a Web.config file, it cannot be dynamically configured in the same way as ETW.</span></span> <span data-ttu-id="cf07b-112">`endToEndTracing` 構成要素を有効にするには、アプリケーションをリサイクルする必要があります。</span><span class="sxs-lookup"><span data-stu-id="cf07b-112">For the `endToEndTracing` configuration element to take effect, the application must be recycled.</span></span>  
  
 <span data-ttu-id="cf07b-113">アクティビティはアクティビティ ID と呼ばれる識別子の交換によって関連付けられます。</span><span class="sxs-lookup"><span data-stu-id="cf07b-113">Activities are correlated by the interchange of an identifier called the activity ID.</span></span> <span data-ttu-id="cf07b-114">この識別子は GUID で、System.Diagnostics.CorrelationManager クラスによって生成されます。</span><span class="sxs-lookup"><span data-stu-id="cf07b-114">This identifier is a GUID, and is generated by the System.Diagnostics.CorrelationManager class.</span></span> <span data-ttu-id="cf07b-115">System.Diagnostics.Trace.CorrelationManager.ActivityID を操作する場合は、実行の制御が WCF コードに転送を返すときに値を元の値に設定するようにしてください。</span><span class="sxs-lookup"><span data-stu-id="cf07b-115">If you manipulate System.Diagnostics.Trace.CorrelationManager.ActivityID, ensure that the value is set to original when execution control transfers back to WCF code.</span></span>  <span data-ttu-id="cf07b-116">また、非同期 WCF プログラミング モデルを使用する場合は、System.Diagnostics.Trace.CorrelationManager.ActivityID がスレッド間で転送されるようにしてください。</span><span class="sxs-lookup"><span data-stu-id="cf07b-116">Also, if you use an asynchronous WCF programming model ensure that System.Diagnostics.Trace.CorrelationManager.ActivityID is transferred between the threads.</span></span>  
  
## <a name="message-flow-tracing-and-rest-services"></a><span data-ttu-id="cf07b-117">メッセージ フローのトレース REST サービス</span><span class="sxs-lookup"><span data-stu-id="cf07b-117">Message Flow Tracing and REST Services</span></span>  

 <span data-ttu-id="cf07b-118">メッセージ フローのトレースでは、要求をエンドツーエンドでトレースできます。</span><span class="sxs-lookup"><span data-stu-id="cf07b-118">Message flow tracing allows you to trace a request end to end.</span></span>  <span data-ttu-id="cf07b-119">SOAP ベースのサービスでは、アクティビティ ID は、SOAP メッセージのヘッダー内で送信されます。</span><span class="sxs-lookup"><span data-stu-id="cf07b-119">With SOAP-based services an Activity ID is sent in a SOAP message header.</span></span> <span data-ttu-id="cf07b-120">REST 要求にはこのヘッダーは含まれないため、代わりに、特殊な HTTP イベント ヘッダーが使用されます。</span><span class="sxs-lookup"><span data-stu-id="cf07b-120">REST requests do not contain this header so a special HTTP event header is used instead.</span></span> <span data-ttu-id="cf07b-121">次のコード スニペットには、アクティビティ ID をプログラムで取得する方法が示されています。</span><span class="sxs-lookup"><span data-stu-id="cf07b-121">The following code snippet shows how you can programmatically retrieve the Activity ID value:</span></span>  
  
```csharp
Object output = null;
if (OperationContext.Current.IncomingMessageProperties.TryGetValue(HttpRequestMessageProperty.Name, out output))
{
   HttpRequestMessageProperty httpHeaders = output as HttpRequestMessageProperty;
   // Retrieve the Activity Id from the HTTP header    string e2eId = httpHeaders.Headers["E2EActivity"];
   // ...
}
```

 <span data-ttu-id="cf07b-122">次のコードを使用するとプログラムでヘッダーを追加できます。</span><span class="sxs-lookup"><span data-stu-id="cf07b-122">You can programmatically add the header using the following code:</span></span>  
  
```csharp  
HttpContent content = new StreamContent(contentStream);  
Guid correlation = Guid.NewGuid();  
content.Headers.Add("E2EActivity", Convert.ToBase64String(correlation.ToByteArray()));  
```
